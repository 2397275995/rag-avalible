### 1. 目的与问题定义

检索增强生成（RAG）系统的“幻觉”通常来自两类机制失效：

- **外部上下文复制失败**：模型未能将检索到的证据正确复制到输出中（copy heads 未有效工作或信息在残差流中被稀释）。
- **参数知识过度注入**：模型在后层过度依赖参数化知识（knowledge FFN 在残差流中占主导），导致输出与外部证据不一致。

本实验目标是实现一个轻量、可解释、无需微调的统一框架：

- **检测**：在 span/chunk 粒度上输出幻觉风险概率，并给出可解释特征贡献。
- **缓解**：生成时当风险过高，进行电路级重加权，降低幻觉率，且不更新模型参数。
- **解释**：同时提供电路解释（哪些层/头/FFN）与稀疏特征解释（哪些单义特征触发风险）。

### 2. 方法总览（MechSparse 管线）

MechSparse 将 ReDeEP 的电路洞见与 RAGLens 的 SAE 特征学习融合为以下五段流程（离线训练 + 在线推理/缓解）：

1. **ReDeEP chunk-level 打分**：对每条回答切分 span/chunk，生成外部复制分数与参数知识分数，并附带幻觉标签。
2. **电路识别（复用 ReDeEP 思路）**：从 chunk-level 分数中筛选复制头集合 𝒜 与知识 FFN 集合 𝓕，并构造 ReDeEP 代理变量 H。
3. **电路引导残差构造**：仅提取 𝒜/𝓕 对残差流的贡献，形成 token 级向量 r，并按两类电路拼接为 r = \[r_ext; r_par]。
4. **电路条件 SAE（SAE on r）**：在 r 上训练/加载 SAE，得到稀疏激活特征 z；对输出 token span 做池化得到样本级稀疏特征。
5. **条件互信息筛选 + 可解释加性模型（GAM/EBM）**：计算 I(z; label | H) 选择少量关键特征，训练可解释加性模型输出风险概率；在线时用于触发缓解策略。

缓解阶段为 **AARF+**：当风险超过阈值 τ 时，根据当前激活的 SAE 特征与其对电路贡献的归因，选择电路子集进行重加权（增强复制、抑制参数注入）。

### 3. 代码组织与关键产物

本仓库包含三个主要子目录：

- **`ReDEeP-ICLR/`**：ReDeEP 基线与 AARF 相关脚本（本实验在此基础上新增/改造 MechSparse 抽取与 AARF+ 原型）。
- **`RAGLens/`**：SAE 特征提取与可解释检测器（EBM/GAM）框架（本实验新增 MechSparse 编码与条件互信息流程）。
- **`MechSparse/`**：集成说明、实验指南与本技术文档。

关键产物（文件级）：

- **电路与训练数据（ReDeEP 侧导出）**
  - `mechsparse_circuits.json`：复制头集合 𝒜（layer/head 列表）与知识 FFN 集合 𝓕（layer id 列表）及映射信息与筛选参数。
  - `mechsparse_redeep_chunk_dataset.jsonl`：每条 span 一行，包含 `input/output/label/H/split` 等字段（训练/评估 detector 的统一输入）。

- **SAE（MechSparse SAE）**
  - `mechsparse_sae.pt`（或其他 checkpoint）：以 r 的维度作为输入的 SAE 模型参数（输入维度约为 `2*hidden_size`）。

- **检测器（RAGLens/MechSparse Detector）**
  - `mechsparse_detector.pkl`：保存已训练的可解释加性模型、top 特征索引、以及电路参数（运行时不包含 tokenizer/LLM/SAE 本体）。

### 4. 数据与标注约定

本实验面向多个数据集与任务场景：

- **RAGTruth**：包含多任务子集（Summary/QA/Data2txt），并提供 span 级或近似可用的幻觉标注信息。
- **Dolly(AC)**、**AggreFact**、**TofuEval**：用于跨域/不同标注风格下的泛化评估。

在工程实现中，ReDeEP 的 chunk-level 输出负责把“原始 response + spans + 标签”转化为统一的 span 级结构化记录，使后续电路选择与 detector 训练可以统一进行。

---

### 5. 阶段 A：ReDeEP chunk-level 打分（输入输出与意义）

该阶段对每条 response 切分为多个 chunk/span，并为每个 span 生成两类分数：

- **外部复制相关分数（external / prompt_attention_score）**：反映模型在生成该 span 时，是否将注意力集中到对应的上下文证据，并形成“有效复制”的通路信号。
- **参数知识相关分数（parameter / parameter_knowledge_scores）**：反映模型在不同层通过参数知识推高某些输出分布的程度（近似 LogitLens/分布差异度量）。

同时，记录该 span 是否被标注为幻觉（`hallucination_label`）。这为后续：

- 电路筛选（哪些 head/层与幻觉显著相关）
- 代理变量 H 的构造（外部与参数两类信号的对抗）
提供监督信号。

注意：该阶段依赖具体模型与数据集的预处理，输出路径与字段结构需保持一致，才能进入后续导出阶段。

---

### 6. 阶段 B：电路导出（𝒜、𝓕 与 H）

该阶段从 chunk-level 输出中筛选电路集合并构造代理变量 H。

#### 6.1 复制头集合 𝒜（Copy Heads）
筛选规则直觉：复制头的外部复制信号越强，幻觉概率越低。工程实现中使用相关性阈值筛选：

- 对外部复制特征与标签（0/1 幻觉）计算相关性，挑选与“非幻觉”负相关/或与幻觉显著负相关的 head。

#### 6.2 知识 FFN 集合 𝓕（Knowledge FFNs）
筛选规则直觉：后层 FFN 的参数注入信号越强，幻觉概率越高。工程实现中使用：

- 仅保留后层（例如层号不小于某阈值）  
- 对参数知识特征与幻觉标签计算相关性，挑选显著正相关的层。

#### 6.3 ReDeEP 代理变量 H
H 作为“电路层面失效方向”的一个连续代理变量，用于后续条件互信息筛选。直觉形式为：

- 参数注入项（越大越危险）减去外部复制项（越大越安全）  
并进行归一化，以保证不同样本间可比。

#### 6.4 导出数据格式
导出的 `mechsparse_redeep_chunk_dataset.jsonl` 对每个样本至少包含：

- `input`：prompt（可包含 chat template 后的有效输入）
- `output`：当前 span/chunk 的输出文本
- `label`：0/1 幻觉标签
- `H`：代理变量（float）
- `split`：train/test（或其他划分）

该 JSONL 是后续 SAE 训练与 detector 训练的统一入口。

---

### 7. 阶段 C：电路引导残差 r 的构造

MechSparse 的关键创新在于：**不对完整隐藏状态学习稀疏特征，而仅对“幻觉关键电路”的残差贡献学习稀疏特征**。

#### 7.1 外部复制残差 r_ext
对复制头集合 𝒜 中的每个 (layer, head)，重建该 head 的输出贡献并投影到模型维度，再对所有选定 head 求和，得到每个 token 的 r_ext。

#### 7.2 参数注入残差 r_par
对知识 FFN 集合 𝓕 中的每个 layer，提取 FFN 的输出投影（down_proj）对残差流的贡献，并对选定层求和，得到每个 token 的 r_par。

#### 7.3 拼接为 r
对每个 token，形成：

- r = \[r_ext; r_par]  

其中前半段对应外部复制通路、后半段对应参数注入通路。该拼接使 SAE 字典学习具备明确语义方向，方便后续解释与干预。

工程补充：为支持缓解阶段的“特征引导电路选择”，实现还支持输出“分量级贡献”（例如每个 head 或每个 FFN layer 对当前 token 的贡献向量）。

---

### 8. 阶段 D：电路条件 SAE（在 r 上训练或加载）

#### 8.1 训练目标
在 r 上训练 SAE，目标包括：

- **重构损失**：保证稀疏表示仍能较好表达电路残差信息；
- **稀疏正则**：保证每个 token 只激活少量特征，从而提升特征单义性与可解释性。

#### 8.2 输入维度与注意事项
因为 r 是拼接向量，输入维度是 `2 * hidden_size`。这要求：

- SAE 的输入维度与模型 hidden_size 严格对齐；
- 训练与推理使用相同的电路集合定义（𝒜/𝓕）与相同的残差构造方式。

#### 8.3 样本级特征（池化）
检测器训练使用样本级特征，需要把 token 级激活聚合为样本级向量。常用策略：

- 在输出 token span 上做 max-pooling（对每个特征取最大激活），对“概念触发”更敏感。

---

### 9. 阶段 E：条件互信息筛选 + 可解释加性模型（GAM/EBM）

#### 9.1 为什么要做条件互信息
若只做普通互信息 I(z; label)，特征可能学到与 H 强相关但“重复”的信息，降低对电路失效机制的对齐程度。引入条件互信息：

- I(z_k; label | H)  
强调“在固定电路代理 H 的条件下，特征对标签仍提供增量信息”。

工程实现使用分箱近似：将 H 按分位数分箱，在每个 bin 内估计 I(z; label)，再按 bin 频率加权求和。

#### 9.2 选择 top-K′ 特征
在条件互信息得分上排序，选择较小的 K′（例如 64），相比原始 RAGLens 可显著减少特征数量，降低过拟合并提高解释密度。

#### 9.3 可解释加性模型（GAM/EBM）
使用无交互的可解释 boosting 加性模型：

- 输出 `p(hallucination)` 概率；
- 可分解每个特征对最终 log-odds 的贡献（形状函数），实现全局与局部解释；
- 可把 H 作为额外输入变量加入模型，使其成为强协变量（帮助稳健性与可解释对齐）。

训练完成后导出 detector（保存所需的特征索引、电路参数与分类器本身）。

### 10. 在线缓解：AARF+（特征引导的电路子集重加权）

#### 10.1 触发机制
生成过程中每一步对当前 prompt + 已生成 suffix 的输出段做风险评估：

- 若 `p(hallucination) > τ`（阈值），则触发干预。

#### 10.2 干预对象
干预仅作用于已识别的两类电路通路：

- 复制电路（copy layers / copy heads 的集合）
- 知识 FFN（knowledge layers 的集合）

#### 10.3 特征引导的子集选择（区别于全局调制）
本实验实现采用“特征引导子集干预”：

- 对最后一个 token 提取电路残差 r，并 SAE 编码得到当前激活特征；
- 将活跃特征与电路分量贡献做归因，选择少量“最相关”的复制层与知识层子集；
- 仅对该子集进行放大/衰减，而不是对所有电路层统一调制。

该设计的好处：

- 更贴近“单义特征—电路”对应关系；
- 干预更精准，避免过度影响其他正常生成通道；
- 解释性更强：可以报告“本次触发由哪些特征导致，对哪些电路子集进行了调制”。

#### 10.4 重加权形式
干预通过对注意力输出与 FFN 输出进行乘法缩放来实现：

- 复制通路：放大（>1）
- 参数注入通路：衰减（<1）

无需微调参数，属于推理期控制策略。

### 11. 评估协议与推荐汇报指标

#### 11.1 检测指标
建议至少报告：

- **AUC**：对阈值不敏感，适合比较检测性能；
- **Acc**：直观准确率；
- **macro-F1**：对类别不平衡更稳健。

建议按数据集与任务类型分别报告，并给出均值与方差（多次种子/子采样）。

#### 11.2 消融实验（建议写入论文）
推荐三组消融：

- **去电路引导**：改为对完整隐藏态编码；
- **去条件互信息**：改为普通互信息特征选择；
- **去 H 协变量**：检测器不接入 H。

#### 11.3 缓解指标
推荐报告：

- 幻觉率（人工或强一致性评估器辅助）；
- 触发率（生成过程中触发干预的步数占比）；
- 触发后转化率（高风险输出转为低风险的比例）；
- 输出质量副作用（是否出现过度保守、重复、无关拒答等）。

### 13. 工程稳定性与环境说明（重要）

#### 13.1 PyTorch/CUDA 兼容性
不同服务器环境可能无法安装带特定 CUDA 后缀的 PyTorch 版本。推荐做法是：

- 使用 PyTorch 官方提供的 cu121/cu124 轮子；
- 在 requirements 中不要写死不可获得的版本后缀；
- 对推理端（AARF+）保持依赖最小化，减少编译型依赖（如 vLLM）带来的环境不确定性。

#### 13.2 SAE 类型兼容
实验中可能同时使用：

- 返回张量激活的简化 SAE（便于快速验证）
- 返回稀疏结构输出的 SAE（含 top-k 与 pre-acts，用于更强稀疏性）

工程实现已做兼容，确保 detector 训练与在线推理都能运行。

### 14. 局限性与未来工作（建议写入论文最后一节）

- **更严格的特征归因**：当前缓解侧的“特征→电路子集”归因是高效近似；未来可引入更严格的梯度积分归因以增强因果解释。
- **长上下文与在线更新**：长上下文下电路贡献估计更难，SAE 可考虑在线更新与滑窗策略。
- **跨模型泛化**：电路集合与 SAE 字典跨模型迁移需要更系统的对齐策略与误差分析。
- **评估一致性**：幻觉标注存在主观性，建议结合自动一致性评估器与人工抽样复核。

### 15. 结论（交付要点总结）

本实验实现了一个统一的、可解释的 RAG 幻觉检测与缓解管线：

- 通过 ReDeEP 的电路筛选得到关键电路集合 𝒜/𝓕 与代理变量 H；
- 仅对电路残差贡献构造表示 r，并在其上学习 SAE 稀疏特征；
- 使用条件互信息筛选与可解释加性模型输出风险概率与解释；
- 在线触发 AARF+ 干预，采用特征引导的电路子集重加权，降低幻觉风险且无需微调。

该框架兼具轻量、可解释与可扩展性，适合用作论文“机制可解释检测 + 推理期控制缓解”的完整实验基线与展示系统。
